Data Engineering Pipeline

Tools used: Docker, Nifi, Spark, MySQL, Grafana

Objective

To undertake a small pipeline development project that considers core elements of the data engineering lifecycle (ingestion, transformation and serving). Investigate and report on the design and implementation of an end-to-end data processing system that can serve data analytics/ML outcomes to address a particular challenge.

Results

Our data pipeline aimed to develop a model predicting the salary of United States citizens based on several factors, including gender, age, education, and marital status. Additionally, we created a dashboard to visualise trends within the U.S. To achieve this, we sourced sample data from Census Microdata, which is accessible through the official government website's API. In the image below, you can see how users were able to modify various variables to view predictions for different scenarios.

The data pipeline begins with Apache Nifi, where JSON-formatted data is ingested, transformed, and then stored in a MySQL database. Apache Spark accesses this database to train a machine learning model. Additionally, we've connected the database to Grafana, where we've built a dashboard to visualize compelling insights from the data.
