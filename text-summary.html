Automated Text Summarisation

Tools used: Python (beautifulsoup, nltk, tkinter, translation)

Objective

The aim of this project is to perform extractive text summarization in Python. The input text for summarization can be in the form of a text file or a URL to a website in any language (we also perform translation). After generating a concise summary, we want to visualize the information within the summary.

Results

The first thing that shows up when we run the program is the GUI presented below. We offer many customisable parameters on the upper right and left corner that affect the size of the summary. In the middle top, we have a place to paste our URL to article that we want to summarise. Alternatively, we can paste our text into a large textbox and press the relevant button.

To illustrate text summarization in action, we present an example of a summary of a BBC article (link). This type of summarization, known as extractive summarization, selects sentences directly from the original article without generating new ones. The selected sentences are those that contain the most frequently occurring words from the entire article. In the interface below, you can view the generated summary in the main text box. Other options include viewing the full article or a translated version.

The generated summary is also automatically saved to a .txt file in the same directory. Additionally, you can create a word cloud of the most frequently used words by simply clicking a button.
